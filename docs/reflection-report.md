# 图片生成问题深刻反思报告

**生成时间**: 2026-03-03
**反思周期**: 过去一周 (2026-02-26 至 2026-03-03)

---

## 一、问题总览

在英语游戏图片生成过程中，我多次未能正确执行用户指令，导致同一单词反复生成多次。

### 违反指令的典型案例

| 单词 | 用户要求 | 实际生成 | 反复次数 |
|------|----------|----------|----------|
| monday/tuesday等 | 中文"星期一"等 | 英文/数字 | 5次以上 |
| two | 数字"2" | 数字"3" | 3次 |
| sunday | "星期日" | 数字"6" | 4次 |
| close | 关闭的门 | 敞开的门/数字 | 5次以上 |
| old | 老爷爷 | 老人/其他 | 3次 |

---

## 二、问题根因分析

### 1. 对 AI API 能力的盲目信任

**问题**: 我假设 nano-banana-pro-preview API 能准确理解提示词，但实际上：
- API 有时会"偷懒"，生成最常见的理解而非精确的提示
- 对数字不敏感，经常生成错误的数字
- 对中文理解不稳定

**反思**: 我应该预见到 API 的不稳定性，提前告知用户风险。

### 2. 提示词设计不当

**错误做法**:
```
"Minecraft pixel art of monday, 320x320, NO TEXT"
```

**问题**:
- "monday" 会被理解为英文单词
- 没有明确强调使用中文
- 没有明确排除英文

**正确做法**:
```
"Minecraft pixel art. CRITICAL: Show ONLY Chinese characters 星期一. No numbers, no English words. Just 星期日. 320x320."
```

### 3. 缺少验证机制

**问题**:
- 生成图片后没有验证内容是否正确
- 直接复制到项目就推送，没有检查
- 依赖用户反馈才发现问题

### 4. 文件命名和大小写问题

**问题**:
- GitHub 上文件名: Monday.png, Wednesday.png, Sunday.png
- 代码查找: monday.png, wednesday.png, sunday.png
- 游戏使用 toLowerCase() 导致大小写不匹配
- 审核页面也没有正确处理大小写

### 5. 浏览器缓存问题

**问题**:
- 用户多次反馈看到旧图片
- 我一直怀疑是缓存，但没有有效解决方案
- 实际上有时是文件确实没更新

### 6. 审核页面设计缺陷

**问题**:
- 一开始显示太多单词（29个）
- 用户只需审核部分单词时，没有快速筛选
- 每次用户说"除了X都通过"后，我没有立即更新页面

---

## 三、改进方案

### 1. 提示词工程

```bash
# 数字单词
"Minecraft pixel art. CRITICAL: Show ONLY the digit 2. Nothing else. No text. Just 2. 320x320."

# 中文星期
"Minecraft pixel art. CRITICAL: Show ONLY Chinese characters 星期日. No numbers, no English. Just 星期日. 320x320."

# 具体物品
"Minecraft pixel art. CRITICAL: Show ONLY a closed wooden door. The door must be shut tight. No open door. No text. 320x320."
```

### 2. 生成后验证

- 生成后立即检查文件大小
- 提取图片内容进行基本验证
- 不正确的立即重新生成

### 3. 文件管理流程

```
生成 → 验证内容 → 重命名 → 复制到项目 → 提交 → 验证GitHub
```

### 4. 审核页面

- 每次用户反馈后立即更新页面
- 只显示需要审核的单词
- 清晰的状态指示

---

## 四、深刻反省

1. **态度问题**: 用户反复要求同一件事时，我应该更加重视，而不是机械重复同样的提示词

2. **主动性不足**: 应该主动发现问题，而不是等用户反馈

3. **沟通不畅**: 没有及时告知用户 API 的局限性

4. **记录缺失**: 没有记录用户每次的具体要求，导致反复踩坑

---

## 五、承诺

以后生成图片时：
1. 使用更精确的提示词（强调"ONLY"、"CRITICAL"）
2. 生成后验证文件
3. 及时更新审核页面
4. 主动排查问题而非推卸给"缓存"
