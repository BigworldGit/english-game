# 英语学习游戏反馈机制与素材需求文档

更新时间：2026-04-02

## 1. 设计目标

本游戏面向小学生，反馈机制不能只强调“对错”和“分数”，而应同时满足四个目标：

1. 让学生立刻知道自己答得对不对。
2. 让学生知道自己“学会了什么”和“还没学会什么”。
3. 让学生在答错时不挫败，在答对时有成就感。
4. 让学生愿意继续下一题、下一关、下一天。

核心原则：

- 低惩罚，高引导
- 短反馈，强感知
- 多鼓励“进步”，少强调“失败”
- 成就设计服务学习，不做空洞装饰

## 2. 小学生心理特点与设计原则

### 2.1 需要即时肯定

小学生对延迟反馈不敏感，更依赖“我刚刚做得怎么样”的即时信号。

设计要求：

- 每题作答后 0.2 秒内出现明确反馈
- 答对时给出音效、颜色、轻动画三重确认
- 答错时也给出明确引导，但避免刺耳、夸张的失败表现

### 2.2 容易因连续答错失去动力

如果连续几题只收到“错了”，学生会很快转入逃避状态。

设计要求：

- 错误反馈强调“这题待复习”，而不是“你不行”
- 连续答错 2-3 题后，文案转为支持型，例如“这组词有点像，我们再看一遍”
- 结果页必须给出可执行下一步，而不是只给分数

### 2.3 对视觉和节奏高度敏感

小学年龄段对鲜明视觉、拟游戏化反馈、可预测节奏更容易投入。

设计要求：

- 正确反馈：亮、短、脆、上升感
- 错误反馈：轻、稳、可恢复
- 页面信息层次简单，不同时出现太多提示

### 2.4 需要“我在进步”的证据

学生持续学习的动力，很多来自“我越来越厉害了”的感受。

设计要求：

- 用“已掌握词数”“连续学习天数”“错题修复数”作为长期反馈
- 关卡结束页展示“本关新掌握词”和“待复习词”
- 成就系统优先奖励坚持、修复、积累，而不是单次高分

## 3. 用户反馈机制设计

## 3.1 即时反馈（题目级）

### 正确反馈

目标：让学生马上感受到“答对了，而且值得继续”。

反馈组合：

- 正确选项高亮为绿色
- 正确选项轻微放大后回弹
- 出现短文案：`答对了`
- 右侧状态更新为：`本题已掌握`
- 播放短促上升音效
- 进度条中当前题变为“已掌握”样式

文案风格要求：

- 简短、明确、不夸张
- 推荐：`答对了`、`很好`、`继续`
- 不推荐：过于幼稚或过于吵闹的文案

### 错误反馈

目标：让学生明确知道错在哪，但不产生明显挫败。

反馈组合：

- 错误选项变红
- 正确选项同时变绿
- 出现短文案：`正确答案是：xxx`
- 右侧状态更新为：`本题待复习`
- 播放短促下降音效，但音量和长度都要轻
- 当前词自动加入本关待复习列表

文案风格要求：

- 强调“下一步可修复”
- 推荐：`这题先记下来`、`正确答案是 xxx`
- 不推荐：`你错了`、`太可惜了`

### 即时学习提示

每题反馈后显示一条很短的学习提示：

- `正确答案：sun`
- `这题主要看太阳，不是颜色`
- `这题表示“告别”，不是普通打招呼`

第一版可以用规则模板生成，后续再做人工文案库。

## 3.2 过程反馈（单关进行中）

右侧状态面板建议显示：

- 当前题号：第 x / 10 题
- 当前正确率
- 已答 / 正确 / 错误
- 连续答对题数
- 本关待复习词数
- 当前题状态：待作答 / 已答对 / 已答错 / 回看

目标：

- 让学生知道自己在推进
- 让学生看到错误不会“消失”，而是会进入待复习

## 3.3 关卡结束反馈（单关总结）

结果页不应只显示分数，应回答三个问题：

1. 这关表现如何？
2. 我掌握了哪些词？
3. 下一步应该做什么？

推荐模块：

- 正确数 / 错误数 / 正确率
- 本关新掌握词数
- 本关待复习词数
- 本关连续答对峰值
- 本关待复习词列表
- 下一步按钮：`复习错题` / `再练一次` / `进入下一关`

结果文案按区间区分：

- 90%-100%：`这关掌握得很稳，可以进入下一关`
- 70%-89%：`大部分词已经掌握，建议先复习错题`
- 50%-69%：`有一些词容易混淆，先再练一次更合适`
- 0%-49%：`这关还不稳，先把待复习词看一遍`

## 3.4 长期反馈（学习档案与成就）

建议长期展示以下指标：

- 累计掌握词数
- 当前年级掌握进度
- 连续学习天数
- 累计修复错题数
- 最长连续答对记录

成就系统建议只保留真正有学习意义的类型：

- 完成第一关
- 连续答对 10 题
- 累计掌握 50 / 100 / 200 个词
- 连续学习 3 / 7 / 14 天
- 修复 10 个曾经答错的词
- 某一关全对

注意：

- 徽章数量不要太多
- 每个徽章都要有清晰达成条件
- 徽章获取必须伴随清晰音效和视觉奖励

## 4. 音效设计需求

目标：

- 提升作答节奏感
- 增强“答对”的奖励感
- 降低“答错”的挫败感

风格要求：

- 像素风、清脆、短促
- 避免真实乐器长音和复杂旋律
- 优先游戏 UI 音效风格

音效列表：

### 4.1 即时答题音效

1. `sfx-answer-correct.mp3`
- 用途：答对
- 风格：上升音阶、明亮、脆
- 时长：0.4-0.8 秒

2. `sfx-answer-wrong.mp3`
- 用途：答错
- 风格：轻下降音，不刺耳
- 时长：0.3-0.6 秒

3. `sfx-select-option.mp3`
- 用途：点击选项
- 风格：轻木块/像素按钮声
- 时长：0.1-0.2 秒

4. `sfx-hover-ui.mp3`
- 用途：悬停按钮或聚焦
- 风格：极轻、可选
- 时长：0.05-0.1 秒

### 4.2 关卡反馈音效

5. `sfx-level-complete.mp3`
- 用途：关卡完成
- 风格：短胜利旋律
- 时长：1.0-1.8 秒

6. `sfx-perfect-level.mp3`
- 用途：本关全对
- 风格：更亮、更完整的胜利短乐句
- 时长：1.5-2.5 秒

7. `sfx-review-unlock.mp3`
- 用途：进入错题复习
- 风格：提示型、开启感
- 时长：0.6-1.0 秒

### 4.3 成就音效

8. `sfx-achievement-unlock.mp3`
- 用途：解锁徽章/成就
- 风格：宝箱/升级感
- 时长：1.2-2.0 秒

9. `sfx-streak-up.mp3`
- 用途：连续答对提升
- 风格：短促升级音
- 时长：0.4-0.8 秒

## 5. 动画设计需求

目标：

- 让对错反馈和进度成长更有感知
- 不影响阅读和作答节奏

总体要求：

- 单次动画 0.25-0.8 秒为主
- 避免连续大幅度抖动
- 低年级用户优先“清晰”而不是“炫”

动画列表：

### 5.1 作答动画

1. 正确反馈动画
- 元素：正确按钮
- 效果：轻微放大、发光、回弹
- 时长：0.35-0.5 秒

2. 错误反馈动画
- 元素：错误按钮
- 效果：轻微水平抖动一次
- 时长：0.25-0.35 秒

3. 正确答案揭示动画
- 元素：正确按钮
- 效果：短暂绿色描边亮起
- 时长：0.3-0.5 秒

### 5.2 进度动画

4. 当前题完成动画
- 元素：进度条中的当前题块
- 效果：翻转或填色
- 时长：0.3 秒

5. 连续答对动画
- 元素：右侧连续答对数字
- 效果：数字轻跳一下
- 时长：0.25 秒

### 5.3 结果页动画

6. 结果页入场
- 元素：结果面板
- 效果：缩放淡入
- 时长：0.35-0.5 秒

7. 正确率圆环动画
- 元素：圆环
- 效果：从 0% 填充到目标值
- 时长：0.8-1.2 秒

8. 徽章解锁动画
- 元素：徽章卡片
- 效果：弹出 + 微发光
- 时长：0.6-1.0 秒

## 7. 素材生成统一要求

### 7.1 风格要求

- Minecraft-inspired
- voxel pixel art
- blocky
- bright educational game UI
- child-friendly

### 7.2 输出要求

- 优先 PNG
- 漂浮/叠加素材必须透明背景
- 不能带文字水印
- 不要复杂人物面孔
- 不要写实摄影风
- 不要赛博朋克、霓虹紫、恐怖感

### 7.3 视觉感受要求

- 颜色明亮，但不能刺眼
- 对“正确”使用绿色/金色
- 对“错误”使用红色，但要柔和
- 成就类要有“宝物/升级”感觉
- 反馈素材要在手机和桌面都清晰
